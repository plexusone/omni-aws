## [v0.1.0] - 2026-04-25

### Highlights

- Initial release with S3-compatible backend for omnistorage-core

### Added

- S3 backend implementing `omnistorage.ExtendedBackend` interface ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- Support for AWS S3, Cloudflare R2, MinIO, Wasabi, DigitalOcean Spaces ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- Streaming read/write with multipart upload support ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- Server-side copy and move operations ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- Range read requests for partial content ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- MD5 hash support via ETag ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- Configurable concurrency and part size ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- Environment-based and programmatic configuration ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- Backend registry integration (auto-registers as `s3`) ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))

### Dependencies

- AWS SDK v2 for S3 operations
- omnistorage-core v0.3.0 for backend interfaces

### Tests

- Unit tests for config validation, parsing, and interface compliance ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
- Integration tests for S3 operations (require OMNISTORAGE_S3_TEST_BUCKET) ([`7ff260a`](https://github.com/plexusone/omni-aws/commit/7ff260a))
